# 状态词汇与完成口径规范

本文件用于统一 `db-cookbook` 的完成口径，避免“章节存在”“脚本通过”“事实初核”“项目跑通”“出版定稿”被混成一句话。

机器可读版本见 [status-language-contract.json](status-language-contract.json)。

## 唯一状态源

- 内容完成度：`docs/completion-audit.md`
- 目标拆解：`docs/objective-coverage-map.md`
- 项目实战运行状态：`project-workbench/project-manifest.json`
- 事实核查状态：`docs/fact-check-matrix.md`

## 固定词表

| 场景 | 允许用词 | 含义 |
| --- | --- | --- |
| 自动化脚本通过 | `已验证` | 只表示脚本或结构检查通过，不表示出版最终质量成立 |
| 事实核查当前阶段 | `已初核` | 表示已有官方来源或安全资料初步核查，仍需出版级闭环 |
| 仍未闭环 | `待补` | 表示还缺可运行验证、章节留痕、出版级复核或人工编辑 |
| 禁止对外宣称 | `不可宣称` | 表示当前证据不足，不能写成“已完成”“已最终关闭”等口径 |

## 当前边界

- 书稿已形成完整路线。
- 自动验证已通过。
- 项目实战大多仍是可检查骨架，不是端到端可运行工程。
- 事实核查为初核，不是出版最终关闭。

## 不允许的全局说法

- “全书已经达到出版最终定稿”
- “事实核查已全部完成”
- “七个项目已经是完整可运行工程”
- “七个项目已经端到端验证完成”
- “所有项目均已跑通”
- “事实核查已最终关闭”
