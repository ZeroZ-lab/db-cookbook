## 官方依据摘要

- PostgreSQL 官方 WAL 文档说明 WAL 的核心目的是数据完整性和崩溃恢复：数据文件变更必须在对应 WAL 记录刷到持久存储之后才写入。
- Logical Decoding 官方文档说明 PostgreSQL 提供把 SQL 修改流式输出给外部消费者的基础设施，变更流由 logical replication slot 标识，输出格式由 output plugin 决定。
- Logical Decoding Concepts 说明 Logical Decoding 会把 WAL 中偏存储层的变更解码成应用可理解的形式；logical slot 表示可以按原始顺序重放给客户端的变更流。
- 官方文档同时提示 slot 独立于连接持久存在、只在正常操作下发出一次变更，但崩溃后可能从更早 LSN 重发，客户端需要处理重复；slot 也会阻止删除仍需保留的 WAL 或 catalog rows，因此不用的 slot 应删除。
- Logical Replication Architecture 说明 logical replication 先复制初始快照，然后实时发送发布端变更；walsender 启动 WAL 的 logical decoding，pgoutput 将变更转成 logical replication protocol，再按 publication 过滤。
