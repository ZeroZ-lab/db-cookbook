## 官方依据摘要

- Flink 1.19 Timely Stream Processing 文档把 Processing Time 定义为执行算子的机器系统时间，延迟低、性能好，但在分布式和异步环境中不提供确定性。
- Event Time 是事件在生产设备上发生的时间，通常嵌入记录；Event Time 的进展依赖数据而不是机器时钟。
- Watermark 是 Flink 衡量 Event Time 进展的机制。`Watermark(t)` 表示事件时间已到达 `t`，但现实中仍可能出现 timestamp 小于或等于 watermark 的 late elements。
- Event Time 可以在乱序、迟到和历史重放场景中获得更符合事件发生时间的结果，但由于只能有限等待乱序事件，确定性也有边界。
- Window 用来限定无界流上的聚合范围，例如滚动窗口、滑动窗口和会话窗口。
