## 官方依据摘要

- PostgreSQL 17 文档列出 B-tree、Hash、GiST、SP-GiST、GIN、BRIN 等索引类型，并说明默认 `CREATE INDEX` 创建 B-tree。
- B-tree 覆盖常见等值、范围、排序和部分前缀模式匹配场景。
- Hash 只适合简单等值比较。
- GiST、SP-GiST、GIN、BRIN 的可用操作依赖具体索引策略、数据类型和 operator class，不能写成所有场景通用保证。
- BRIN 通过保存连续物理块范围摘要工作，适合与物理存储顺序相关性高的列。
