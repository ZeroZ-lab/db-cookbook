## 官方依据摘要

- PostgreSQL Row-Level Security 文档说明表可以有 row security policies，按用户限制普通查询返回哪些行以及插入、更新、删除哪些行；启用 RLS 后，如果没有策略，会使用 default-deny。
- PostgreSQL 文档同时说明 RLS 与 SQL privilege system 并存，superuser、BYPASSRLS 和表 owner 等存在绕过边界，说明权限模型必须明确主体、资源和绕过条件。
- AWS S3 文档说明 S3 buckets 和 objects 默认私有，资源所有者决定谁可以访问以及允许哪些动作，说明对象存储权限不能假设自动继承数据库权限。
- Qdrant filtering 文档说明可以在搜索或检索 points 时对 payload 和 point id 设置条件，并支持 AND、OR、NOT 等组合；这支持向量检索过滤，但它不是完整授权系统。
- OWASP Authorization Cheat Sheet 强调 least privilege、deny by default、每次请求都校验权限，并建议不要依赖单一框架或组件作为唯一授权控制。
- OWASP LLM Top 10 2025 将 Sensitive Information Disclosure、Vector and Embedding Weaknesses 等列为 LLM 应用风险，支持在 RAG / GraphRAG 链路中显式记录来源、权限、过滤和审计。
