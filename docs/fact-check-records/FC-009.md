## 官方依据摘要

- Doris 3.x 官方文档将 Apache Doris 定位为 MPP-based real-time data warehouse，用于报表分析、Ad Hoc 查询、统一数仓和湖仓查询加速。
- 官方架构文档说明 FE 主要负责用户请求、查询解析和规划、元数据管理、节点管理；BE 主要负责数据存储和查询执行，数据分片并在 BE 节点上多副本存储。
- 官方技术概览说明 Doris 查询引擎基于 MPP 架构，支持节点间和节点内并行执行，并通过列式存储、索引、向量化和 Pipeline 执行优化分析查询。
- 官方表模型文档说明 Doris 支持 Duplicate Key、Unique Key、Aggregate Key 三类模型，分别对应明细保留、唯一键更新 / 去重、按 Key 预聚合。
- Duplicate Key 官方文档说明该模型保留原始数据，不做去重或聚合，适合明细分析。
- Unique Key 官方文档说明该模型保证 Key 唯一，插入重复 Key 时更新旧记录，适合更新型场景。
- Aggregate Key 官方文档说明该模型通过预聚合降低计算冗余和提升查询性能，但只存储聚合数据而非完整原始明细。
