## 官方依据摘要

- pgvector README 说明默认执行 exact nearest neighbor search；添加近似索引会用召回换速度，支持 HNSW 和 IVFFlat。HNSW 使用多层图结构，查询性能通常优于 IVFFlat，但构建更慢、内存更多；IVFFlat 把向量分到 lists 中，查询时搜索一部分 lists，需要数据后建索引并调 lists / probes。
- Milvus index 文档列出 floating-point embeddings 支持 FLAT、IVF_FLAT、IVF_PQ、IVF_SQ8、HNSW、HNSW_SQ、HNSW_PQ、HNSW_PRQ、SCANN 等索引，并按场景说明 FLAT 适合小数据和 100% recall，IVF_PQ / SQ 等量化型索引适合资源受限但接受轻微召回损失的场景。
- Milvus vector index 文档说明创建索引时需要指定 field、index_type、metric_type 和 index-specific params，说明索引选择依赖字段、度量和参数。
- Qdrant indexing 文档说明 vector index 加速向量搜索，payload index 加速过滤；dense vector index 当前使用 HNSW，并强调过滤场景需要 vector index 与 payload index 配合。
- Qdrant quantization 文档说明量化用于压缩高维向量、降低内存、加速搜索，但会引入近似误差，形成准确率、存储效率和搜索速度之间的权衡。
