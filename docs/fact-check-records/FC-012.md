## 官方依据摘要

- Delta Lake 官方文档将 Delta Lake 定位为在现有数据湖之上构建 Lakehouse 架构的开源项目，提供 ACID transactions、scalable metadata handling，并统一 streaming 和 batch data processing。
- Delta Lake 官方文档还列出 schema enforcement、time travel、upserts / deletes、Change Data Feed 和多连接器生态，说明它是湖仓表格式 / 存储层能力，不是完整业务治理系统。
- Delta Lake batch 文档说明 Delta table 可以由 metastore 或路径定义，支持 batch reads/writes、time travel、schema validation、schema update 和 table metadata。
- Hudi 官方 overview 将 Hudi 的核心概念概括为 Timeline、File Layout、Table Types、Query Types。
- Hudi table types 文档说明 CoW 适合读多场景，更新或删除会产生新的 base files；MoR 通过 log files 和周期性 compaction 平衡写入与读取，并支持近实时可用。
- Hudi 文档说明 snapshot queries、incremental queries、read-optimized queries 是不同读取语义，不能把 Hudi 简化成“只有一种表读法”。
