## 官方依据摘要

- Neo4j graph database stores data as nodes, relationships, and properties，而不是表或文档。
- Node 表示实体或离散对象，可以有 labels 和 properties。
- Relationship 连接 source node 和 target node，必须有 start node、end node、direction 和 exactly one type，也可以有 properties。
- Cypher 的核心是 graph pattern matching；`MATCH` 描述要搜索的图模式并返回匹配路径。
- Pattern 是可查询的图结构规格，Path 是图中实际匹配该模式的一组节点和关系序列。
