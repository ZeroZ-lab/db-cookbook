# 小节级内容生成模板

> 基于《数据库烹饪书》第2章第1、2节（2.1、2.2）总结的标准模板

## 📋 模板使用说明

**适用场景**：
- 你需要为任何章节创建一个新的小节
- 你希望保持与第2.1、2.2节相同的详细程度和质量
- 你需要确保内容结构的一致性

**使用方法**：
1. 复制这个模板
2. 根据小节主题填充内容
3. 使用后面的【内容片段库】快速组装
4. 用【质量检查清单】验证

## 🎯 小节结构模板

### 必备的8个部分

```markdown
### X.X 小节标题：副标题

【必选开头段落】
一句话说明这个小节要解决的核心问题，或者与上一小节的关系。

（示例：基础查询解决的是取数边界问题。）

#### 一、为什么/问题的提出（3-5个论点）

**格式**：
- **论点1标题**：详细说明
  - 示例或场景
  - 数据或代码示例
  - 为什么这样不行/为什么需要这样

**要求**：
- 至少3个论点，通常5个左右
- 每个论点要有具体例子
- 用业务场景说明，不要抽象说教
- 总字数：600-900字

#### 二、核心判断（>引导的一句话）

**格式**：
> 核心判断句

**要求**：
- 用>标记
- 一句话概括本质
- 不是定义，而是价值判断
- 总字数：50-100字

**示例**：
> 基础查询不只是"查出来"，而是清楚定义：取哪些字段、取哪些记录、什么顺序、返回多少行

#### 三、详细机制/概念解释（3-8个子点）

**格式**：
##### 3.1 子概念1
- 定义
- 作用
- 示例
- 注意事项

##### 3.2 子概念2
...

**要求**：
- 每个子概念200-400字
- 包含代码示例
- 包含对比说明
- 总字数：800-1500字

#### 四、与其他概念的关系/延伸

**格式**：
- 这个概念在后续章节如何延伸
- 在其他系统中的体现
- 与相似概念的区别

**要求**：
- 连接到后续章节
- 说明知识的迁移性
- 总字数：300-500字

#### 五、核心判断（再次强调）

**格式**：
> 针对本小节的总结性判断

**要求**：
- 与"二、核心判断"不同
- 这是对本小节内容的总结
- 总字数：50-100字

#### 六、常见误区（5个）

**格式**：
**误区X：误区标题**
- **说明**：为什么是误区
- **后果**：会导致什么问题
- **正确理解**：应该怎么理解

**要求**：
- 5个误区
- 每个误区150-250字
- 总字数：750-1250字

#### 七、实战任务（2-3个）

**格式**：
**任务X：任务名称**

给定[场景/条件]，完成以下步骤：
1. 步骤1
2. 步骤2
3. 步骤3

观察指标：
- 指标1
- 指标2

**要求**：
- 2-3个任务
- 任务可操作、可验证
- 包含具体的观察指标
- 总字数：500-800字

#### 八、小结（3-5句话）

**格式**：
本小节完成了...（1句话）
核心要点是...（2-3句话）
下一节将进入...（1句话）

**要求**：
- 总结本小节的核心要点
- 引出下一小节
- 总字数：150-250字
```

## 🎨 内容风格要求

### 1. 标题层级

```markdown
### X.X 小节标题：副标题

#### 一、一级标题（中文数字）
##### 3.1 二级标题（数字+点）
三级标题（如果需要）
```

### 2. 代码示例

**格式**：
```sql
-- 注释说明
SELECT ...
```

**要求**：
- 所有代码都要有注释
- 使用有意义的表名和字段名（如users, orders, products）
- 复杂SQL要添加逻辑说明

### 3. 核心判断

**格式**：
> 核心判断句

**特征**：
- 用>标记
- 一句话
- 不是定义，是价值判断
- 要有洞察力

**示例类型**：
- "XX不是为了YY，而是为了ZZ"
- "XX回答的是...问题"
- "XX的核心判断是..."
- "XX用YY换取ZZ"

### 4. 常见误区结构

每个误区必须包含：
- **误区标题**：简短、明确
- **说明**（2-3句）：为什么这是误区
- **后果**（1-2句）：会导致什么问题
- **正确理解**（1-2句）：应该怎么理解

### 5. 实战任务结构

每个任务包含：
- **任务名称**：清晰
- **给定条件**：明确
- **操作步骤**：3-7步
- **观察指标**：2-5个
- **可选**：验证方法、对比实验

## 💡 提示和技巧

### 技巧1：如何快速写"为什么"部分
使用"三段式"结构：
- **场景描述**："很多人会遇到..."
- **问题呈现**："这会导致..."
- **引出需求**："所以需要..."

### 技巧2：如何快速写"核心判断"
使用这些模板：
- "XX不是为了YY，而是为了ZZ"
- "XX的核心判断是YY"
- "XX用YY换取ZZ"

### 技巧3：如何快速写"常见误区"
使用这些误区类型：
- **误区一：XX就是YY**（定义误区）
- **误区二：XX总是好的**（过度简化）
- **误区三：XX不重要**（忽视重要性）
- **误区四：XX能解决所有问题**（万能论）
- **误区五：XX很简单**（轻视复杂性）

### 技巧4：如何快速写"实战任务"
使用这些任务类型：
- **基础练习**：写一个简单的SQL
- **对比实验**：对比两种做法
- **综合应用**：解决一个实际问题

### 技巧5：保持一致性的要点
- 统一使用电商场景（users, orders, products, events）
- 统一代码风格（有注释、有说明）
- 统一的标题层级（### → #### 一、）
- 统一的核心判断格式（>开头）
