# RAG 权限过滤边界

权限过滤不能只依赖 LLM 提示词，也不能只依赖向量库的最终结果过滤。

## 检索前过滤

- 根据 `collection_id`、`owner_id`、`access_scope` 缩小可见文档集合。
- 只对可见文档的 chunk 做向量召回。
- 防止无权限 chunk 进入候选集、重排器或上下文拼接。

## 检索后校验

- 对 rerank 后的 chunk 再次检查权限。
- 写入 `retrieval_logs` 时记录过滤条件和用户身份。
- 生成回答前只拼接通过权限校验的上下文。

## 仍不能解决的问题

- 权限字段错误写入。
- 文档解析阶段泄漏敏感内容。
- LLM 根据训练记忆生成未授权内容。
- 日志系统泄漏召回文本。

这些问题需要和数据治理、审计日志、脱敏和应用层鉴权一起处理。
