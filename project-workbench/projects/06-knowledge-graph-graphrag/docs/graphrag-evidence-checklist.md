# GraphRAG 证据拼接检查项

## 检查目标

确保 GraphRAG 上下文拼接过程中的证据完整性、来源可追溯性和回答准确性。

## 检查项

### 1. 向量召回片段

| 检查项 | 说明 | 状态 |
| --- | --- | --- |
| 召回数量 | Top-K 结果数量是否合理（3-10） | [待运行时验证] |
| 来源标注 | 每个片段是否有 source_uri | [待运行时验证] |
| 相关性 | 召回片段与问题是否相关 | [待运行时验证] |
| 去重 | 是否有重复片段 | [待运行时验证] |

### 2. 图路径证据

| 检查项 | 说明 | 状态 |
| --- | --- | --- |
| 路径完整性 | 路径是否从问题实体出发 | [待运行时验证] |
| 跳数限制 | 路径跳数是否在合理范围（1-3） | [待运行时验证] |
| 边类型 | 边类型是否与问题语义匹配 | [待运行时验证] |
| 节点属性 | 关键节点属性是否包含 | [待运行时验证] |

### 3. 上下文拼接

| 检查项 | 说明 | 状态 |
| --- | --- | --- |
| 来源去重 | 向量和图证据是否有重叠来源 | [待运行时验证] |
| 优先级 | 证据按相关性排序 | [待运行时验证] |
| 总长度 | 上下文总长度不超过 LLM 限制 | [待运行时验证] |
| 格式一致 | 向量片段和图路径格式统一 | [待运行时验证] |

### 4. LLM 生成

| 检查项 | 说明 | 状态 |
| --- | --- | --- |
| 引用来源 | 回答中是否引用了具体来源 | [待运行时验证] |
| 事实准确 | 回答内容是否与证据一致 | [待运行时验证] |
| 无幻觉 | 回答是否包含证据中没有的信息 | [待运行时验证] |
| 格式正确 | 回答格式是否符合要求 | [待运行时验证] |

## 拼接流程

```text
1. 用户提问
   "Alice 买了什么电子产品？"

2. 向量召回
   - 从 rag.embeddings 检索语义相似的 chunk
   - 返回 Top-3 片段，带 source_uri

3. 图路径查询
   - 识别实体: Alice (User)
   - 查询路径: Alice -[PLACED]-> Order -[CONTAINS]-> Product
   - 过滤: category = 'electronics'
   - 返回路径证据

4. 上下文拼接
   - 向量片段: [片段1, 片段2, 片段3]
   - 图路径: Alice -> Order(1001) -> Product(Mechanical Keyboard)
   - 来源标注: 每个证据带 source_uri

5. LLM 生成
   - 基于拼接上下文生成回答
   - 引用具体来源
```

## 失败归因

| 失败类型 | 原因 | 改进方向 |
| --- | --- | --- |
| 向量召回失败 | embedding 不匹配 | 调整 embedding 模型 |
| 图路径为空 | 实体未在图中 | 检查图数据导入 |
| 路径过长 | 跳数过多 | 限制跳数或调整查询 |
| 上下文过长 | 证据过多 | 截断或重排 |
| 回答幻觉 | LLM 未遵循上下文 | 调整 prompt |
